# Support

## Where To Ask What

| Need | Best place |
| :-- | :-- |
| Bug report | GitHub issue |
| Idea or product direction | GitHub Discussions |
| Usage question | GitHub Discussions / Q&A |
| New store request | Store request issue template |
| Security issue | Private report via [`SECURITY.md`](./SECURITY.md) |

## Product Support

Use GitHub issues for reproducible product defects. product usage questions go to GitHub Discussions / Q&A, product ideas or Compare Preview workflow feedback belong in GitHub Discussions, and new store requests should use the dedicated store request issue template.

## Security Support

Do not use public issues for security issues. Follow the private process described in [`SECURITY.md`](./SECURITY.md).

## Scope

Support covers the public DealWatch repository, its documented product runtime, the public compare preview surface, and the GitHub Pages public entrypoint.
